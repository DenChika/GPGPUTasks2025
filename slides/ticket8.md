## Узкие места в кернеле

Есть два основных узких места: compute bound или memory bound, то есть во что мы упираемся

1. Если мы упираемся в compute bound, то нужно проверить наличие code divergence. Если утилизация >70% ALU FLOPs, то алгоритм compute bound
2. Если упираемся в memory bound, то если утилизация пропускн способности vram >70%, то стоит подумать про переиспользование данных за счёт локальной памяти. Если меньше 70%, то надо проверить 3 момента: не coalesced access, низкая occupancy, экстремальный code-divergence

Чтобы понять узкое место, надо посмотреть на соотношение Gflops к bandwidth
