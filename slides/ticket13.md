## Radix sort

Здесь нужен алгоритм map + prefix + scatter на каждом бите

map - ставим в массиве 1, если элемент равен 0, и наоборот

prefix - считаем префиксные суммы (2мя кернелами)

scatter - если бит нулевой, то записываем в output просто префиксную сумму (это будет индекс). Если бит 1, то складывает текущий индекс (как index - prefix[index]) + индекс (prefix[n - 1]) 

Так мы получили индексы, отсортированных элементов

Соответственно на каждый шаг нужно по 4 кернела, шагов log

Алгоритм memory bound. Ускориться можно, если проверять не 1 бит на этапе map, а сразу несколько (лучший случай 4 бита). А scatter соответственно для вычисления глобального индекса использует префиксную сумму на прошлом шаге (префиксная сумма второго порядка, где суммы для 0000, 0001 и тд)
