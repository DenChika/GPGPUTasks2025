## Узкие места в зависимости от алгоритма

1. Суммирование массивов - memory bound. Одна операция сложения и 3 операции с vram. Можно добиться максимального coalesced, но упремся всё равно в пропускн способность памяти
2. Умножение матриц - memory bound. Хотя мы можем закешировать тайлы размером 32*32, пропорция чтения / операции станет лучше (1:32), всё равно глобально это memory bound (не уверен). Мы можем эту пропорцию ещё увеличить, использовав алгоритм штрассена или виноградова, или например использовать wwma, квантование, тензорныеые ядра и тд
3. Фрактал мандельброта - compute bound. Чистый расчёт упирается в вычисления, так как все данные лежат в регистрах. А визуализация может быть memory bound, так как помимо вычислений, мы должны записать пиксель во vram
4. Трассировка лучей -понятия не имею
