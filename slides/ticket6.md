## code divergence

Так как у одного варпа один instruction pointer на всех лилипутов, их исполнение происходит "вместе". Соответственно может возникнуть ситуация их расхождения. 

Например, есть if, в одно из тел которого заходит один лилипут. Остальные вынуждены его ждать и в холостую исполнять эту инструкцию, вместо того чтобы выполнять какую-то полезную работу. Наличие таких расхождений называется code-divergence

Ситуация усугубляется при наличии множества ветвлений, или еще хуже - рекурсией

Плохой пример:
if (idx%2 == 0) 

Хороший пример:
int idx = get_local_id(1) + get_local_size(1) * get_local_id(0);
// idx = y + 32*x
if (idx % 32 < 16)
    foo();
else
    bar();
