## AoS => SoA

Представим есть структура данных с каким-то переменными. Эти переменные в памяти будут храниться последовательно. Соответственно, если нам нужен доступ к какой-то одной переменной каждой структуры в массиве, то этот паттерн доступа будет не coalesced

Оптимизация заключается в том, чтобы хранить не массив структур, а несколько массивов для каждой из переменных. Тогда доступ к памяти будет последовательным, соответственно coalesced
